# 判作业 · 聚焦原型

只做一件事并做好：**批改作业 → 存结构化数据 → 看进步/退步**。

核心理念：模型不训练（前沿多模态模型已胜任批改），护城河在"每个孩子的纵向数据 + 课标知识点体系 + 趋势分析"。

## 运行

```bash
cd "THE APP"
node server.js          # 零依赖
# 打开 http://localhost:5173
```

## 流程

1. 新建孩子（姓名/年级/年龄/国籍）
2. 上传作业照片 → Gemini 批改（逐题对错、错误类型、知识点、建议）
3. 结果自动存入 `data/records.json`（按孩子）
4. "历史与趋势"页：代码算出各知识点 早期→近期 正确率，LLM 给出进步/退步解读

## 配置（.env）

```
GEMINI_API_KEY=你的key       # ⚠️ 不要分享，已 gitignore
GEMINI_MODEL=gemini-2.5-flash
```

## 关键设计

- **防幻觉**：提示词要求模型"先独立算标准答案，再读学生作答，最后比对"
- **标签一致性**：要求用规范课标口径命名知识点，便于长期统计
- **趋势 = 代码算 + LLM 解读**：正确率趋势用代码计算；"无数据/未涉及"的知识点明确不判为退步（已修过一次误判）

## 待办（做成产品前）

- 真实作业的复杂场景：多页、潦草字迹、应用题、作文（主观题需评分量规）
- 知识点标签用固定枚举约束（现在靠提示词，量大会漂移）
- 本地 JSON → 真数据库；家长一键纠正错判（纠正数据可回流做微调）
- 儿童数据隐私与合规（最大风险）
- 成本测算：当前 key 是计费档，调用会产生费用
